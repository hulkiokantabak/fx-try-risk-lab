# Security Policy

## Supported Version

The current `main` branch is the supported version for security fixes.

## Reporting a Vulnerability

Please do not open a public GitHub issue for a sensitive security problem.

Instead:

1. Contact the repository owner privately through GitHub.
2. Include clear reproduction steps and impact if known.
3. Allow time for review before public disclosure.

For non-sensitive hardening suggestions, a normal GitHub issue is fine.
